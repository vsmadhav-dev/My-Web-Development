Dockerizing a Simple App

Introduction

In the world of modern software development, the phrase "it works on my machine" is a common hurdle. Docker solves this by using containerization technology to package an application and all its dependencies—libraries, configuration files, and runtime—into a single, standardized unit called a container.

This documentation provides a comprehensive walkthrough for "Dockerizing" a simple web application. By the end of this guide, you will understand how to move away from manual environment setups and toward a portable, scalable, and isolated infrastructure.

Learning Objectives

By the end of this guide, you will be able to:

  • Understand the basics of Docker and containerization
  • Create a Dockerfile for a simple web application
  • Build and run a Docker container
  • Manage Docker images and containers

Prerequisites

Before you begin, ensure you have the following:

  • Docker Desktop (or Docker Engine for Linux)
  • A Terminal/CLI (bash, zsh, or PowerShell).
  • Basic Command Line Knowledge (navigating directories and executing scripts).

Project Structure

Before writing any Docker code, it is essential to organize your application files. A clean directory ensures that Docker only copies the files it needs, keeping the final image small.

our project folder should look like this:

my-docker-app/

├── node_modules/ (Generated later)

├── src/

│ └── index.js (Your main app code)

├── package.json (Metadata and dependencies)

├── .dockerignore (Files to exclude)

└── Dockerfile (The build instructions)

Creating a Dockerfile

A Dockerfile is a text file that contains a series of instructions on how to build a Docker image. Each instruction creates a layer in the image, and the final image is a combination of all these layers.

Create a file named Dockerfile (no extension) and add the following:

# Step 1: Use an official base image FROM node:18-alpine

# Step 2: Set the working directory inside the container WORKDIR /app

# Step 3: Copy package files and install dependencies COPY package*.json ./ RUN npm install

# Step 4: Copy the rest of your application code COPY . .

# Step 5: Expose the port the app runs on EXPOSE 3000

# Step 6: Define the command to run your app CMD ["node", "src/index.js"]

Why use alpine? Alpine is a minimal Linux distribution. Using node:18-alpine instead of the full node:18 image can reduce your image size from 900MB to about 160MB.

Running the Container

Now that you have an image, you can launch it as a "container" (a running instance of that image).

Run the following command:

docker run -p 8080:3000 -d --name my-running-app my-first-docker-app

Here's what each part of the command does:

  • -p 8080:3000: Maps port 3000 inside the container to port 8080 on your host machine.
  • -d: Runs the container in detached mode (in the background).
  • --name my-running-app: Assigns a name to the running container for easier management.
  • my-first-docker-app: The name of the image to run.

Troubleshooting & Common Commands

Documentation isn't complete without a "cheat sheet" for when things go wrong.

  • View Running Containers: docker ps
  • Stop a Container: docker stop my-running-app
  • Remove a Container: docker rm my-running-app
  • View Container Logs: docker logs my-running-app
  • Rebuild the Image: docker build -t my-first-docker-app .
